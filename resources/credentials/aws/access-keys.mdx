Access keys let Arklow reach your AWS Account on your behalf. The same credential can be used for any supported AWS source, scale target, destination, or metric source.

## Required Permissions

To use the credential for:

### Sources

- `sqs:ReceiveMessage`
- `sqs:DeleteMessage`

### Destinations

- `sqs:SendMessage`

## Create your Credentials

Create the access key against the IAM user that holds these permissions. The steps are in [Manage access keys for IAM users](https://docs.aws.amazon.com/IAM/latest/UserGuide/id_credentials_access-keys.html).

## Add Credentials to Arklow

<Steps>
  <Step title="Open the credentials page">
    Go to [**Credentials**](https://app.arklow.io/dashboard/credentials) and click **New credential**.
  </Step>
  <Step title="Pick the type">
    Set **Credential Type** to **AWS Credentials**.
  </Step>
  <Step title="Enter the key">
    Paste the **Access Key ID**. Arklow upper cases it as you type. Paste the **Secret Access Key** below it. The secret stays hidden and is encrypted at rest.
  </Step>
  <Step title="Save">
    Click **Create credential**.
  </Step>
</Steps>
