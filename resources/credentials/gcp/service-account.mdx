A service account key lets Arklow reach Google Cloud on your behalf. Supported Pub/Sub sources, Pub/Sub destinations, and Google Cloud scale targets can use it.

## Required permissions

To use the credential for:

### Sources

- `pubsub.subscriptions.consume` (`roles/pubsub.subscriber`)

### Destinations

- `pubsub.topics.publish` (`roles/pubsub.publisher`)

### Scale targets

All Google Cloud managed instance group targets need:

- `compute.instanceGroupManagers.get`

Targets that manage an autoscaler floor also need:

- `compute.autoscalers.get`
- `compute.autoscalers.update`

Targets that set the group's desired replica count instead need:

- `compute.instanceGroupManagers.update`

These permissions cover the [managed instance group read](https://cloud.google.com/compute/docs/reference/rest/v1/instanceGroupManagers/get), [autoscaler update](https://cloud.google.com/compute/docs/reference/rest/v1/autoscalers/patch), and [group resize](https://cloud.google.com/compute/docs/reference/rest/v1/instanceGroupManagers/resize) operations Arklow uses. The same permission names apply to regional groups.

## Create your credentials

Create a service account before getting started. For more information, see [Create service accounts](https://cloud.google.com/iam/docs/service-accounts-create).

Download the JSON key file. You'll paste its contents into Arklow when creating the credential.

## Add credentials to Arklow

<Steps>
  <Step title="Open the credentials page">
    Go to [**Credentials**](https://app.arklow.io/dashboard/credentials) and click **Create credential**.
  </Step>
  <Step title="Pick the type">
    Set **Credential Type** to **GCP Service Account**.
  </Step>
  <Step title="Paste the key file">
    Open the JSON key file you downloaded and paste it into the **Service Account JSON** field.
  </Step>
  <Step title="Save">
    Click **Create credential**.
  </Step>
</Steps>
