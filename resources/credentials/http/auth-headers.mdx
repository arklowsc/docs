Auth headers are HTTP headers that Arklow attaches to every outbound request to a webhook source/destinations or certain HTTP-driven source/destinations. Use them to pass headers for `Authorization` authentications, or otherwise.

## Add Credentials to Arklow

<Steps>
  <Step title="Open the credentials page">
    Go to [**Credentials**](https://app.arklow.io/dashboard/credentials) and click **Create credential**.
  </Step>
  <Step title="Pick the type">
    Set **Credential Type** to **HTTP Auth Headers**.
  </Step>
  <Step title="Add the headers">
    Under **Authentication Headers**, add one row per header. Put the header name on the left, such as `Authorization`, and its value on the right.
  </Step>
  <Step title="Save">
    Click **Create credential**.
  </Step>
</Steps>

## Special Notes

Headers provided here will always overwrite existing headers of the name field name on a destination's header config, if present.
