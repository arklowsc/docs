An HTTP Auth Headers credential stores headers Arklow can attach to supported outbound HTTP connections. Use one for webhook authentication, a protected Prometheus endpoint, or a compatible HTTP scale target.

## Add credentials to Arklow

<Steps>
  <Step title="Open the credentials page">
    Go to [**Credentials**](https://app.arklow.io/dashboard/credentials) and click **Create credential**.
  </Step>
  <Step title="Pick the type">
    Set **Credential Type** to **HTTP Auth Headers**.
  </Step>
  <Step title="Add the headers">
    Under **Authentication Headers**, add one row per header. Put the header name on the left, such as `Authorization`, and its value on the right.
  </Step>
  <Step title="Save">
    Click **Create credential**.
  </Step>
</Steps>

## Header precedence

Credential headers overwrite destination headers with the same exact name and casing. Use consistent casing to avoid combining two values for what the receiver treats as one header. Keep sensitive values in the credential rather than the destination's ordinary header configuration.
