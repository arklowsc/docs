Ingress is the direct path into Arklow over HTTP.

```
POST https://ingress.arklow.io/v1/ingress
POST https://ingress.arklow.io/v1/ingress/{variant}
```

The first form names the `variant` in the body. The second names it in the path and takes the envelope of `payload`, `metadata`, `tags`, and `version`, or the payload itself, raw: a body without a `payload` key is taken whole as the payload.

```bash
curl -X POST "https://ingress.arklow.io/v1/ingress/orders.created?tag.region=eu" \
  -H "X-Arklow-Auth: $ARKLOW_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"order_id": "8471", "total": 129.50}'
```

Any `tag.<key>=<value>` query parameter merges over the body's tags, and the tags select the [lane](/resources/lanes/index) the action dispatches through. The response carries the action's id as `a_id`, already queued. Every accepted POST creates a new action: queue sources deduplicate on the source message, and direct ingress has no equivalent.
