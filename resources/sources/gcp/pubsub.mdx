The Pub/Sub source pulls work from a Google Cloud Pub/Sub subscription. Each message becomes one action: the message data is the payload.

## Credentials

Uses a [GCP Service Account credential](/resources/credentials/gcp/service-account).

## Add Pub/Sub to Arklow

<Steps>
  <Step title="Open the action flow">
    Open the action definition this subscription should create, select **Flow**, and click **Add source**.
  </Step>
  <Step title="Pick the type">
    Set **Source Type** to **Google Pub/Sub**.
  </Step>
  <Step title="Configure the subscription">
    Enter the **Project ID** and **Subscription ID**.
  </Step>
  <Step title="Choose how attributes become tags">
    Leave **Read platform attributes as tags** enabled when Pub/Sub message attributes should travel with the action as tags. Disable it when only the message data should shape the work.
  </Step>
  <Step title="Select the credential">
    Under **Authentication**, select the GCP Service Account credential.
  </Step>
  <Step title="Save">
    Click **Create source**.
  </Step>
</Steps>

## Deduplication

Actions are keyed to the source message, so a Pub/Sub redelivery resolves to the original action.
