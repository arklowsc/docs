The SQS source pulls work from an Amazon SQS queue. Each message becomes one action: the message body is the payload.

## Credentials

Uses an [AWS credential](/resources/credentials/aws/access-keys).

## Add SQS to Arklow

<Steps>
  <Step title="Open the action flow">
    Open the action definition this queue should create, select **Flow**, and click **Add source**.
  </Step>
  <Step title="Pick the type">
    Set **Source Type** to **AWS SQS**.
  </Step>
  <Step title="Configure the queue">
    Enter the **Queue URL** and select a **Region**.
  </Step>
  <Step title="Choose how attributes become tags">
    Leave **Read platform attributes as tags** enabled when SQS message attributes should travel with the action as tags. Disable it when only the message body should shape the work.
  </Step>
  <Step title="Select the credential">
    Under **Authentication**, select the AWS credential.
  </Step>
  <Step title="Save">
    Click **Create source**.
  </Step>
</Steps>

## Deduplication

Actions are keyed to the source message, so an SQS redelivery resolves to the original action.
