A lane is one admission scope at one destination. Actions in the same lane share occupancy, learned limits, pacing, delivery history, and temporary operator advice.

Lane identity is resolved after rules select the destination and assign tags. Two action variants can share a lane when they reach the same destination with the same control tags. The same control tags at two destinations produce two lanes.

[How Arklow works](/fundamentals/overview#lanes-define-who-shares-admission-limits) follows an end-user lane through admission, shared capacity, and scaling.

## Identity modes

| Mode | Lane identity | Use |
|---|---|---|
| **Complete tags** | The complete normalized tag map | The default when every tag combination should have independent control |
| **Partition tag** | One configured tag name and value | Stable customer, tenant, account, region, or workload scopes |

Tag order has no effect on identity. An action with no tags uses the destination's untagged lane.

## Partition behavior

### Missing values

Actions with a missing or empty partition value use the destination's shared untagged lane regardless of the other tags.

Choose a partition tag that is present on every action that requires isolation.

### Other tags

Tags outside the partition remain attached to the action and available to rules, metrics, and observability. The partition value alone determines the lane and its limits.

## Tag sources

Tags can come from:

- The `tags` field sent to [HTTP ingress](/resources/sources/arklow/ingress).
- `tag.<key>` query parameters on HTTP ingress.
- Source attributes configured to become tags.
- An [Assign Tags rule](/resources/rules/index#assign-tags).

## Limits and advice

Each lane carries the running and unsettled occupancy described in [Admission control](/fundamentals/admission-control#running-and-unsettled-capacity). Admission applies the lane's current limits and any tighter quota from its [capacity pool](/resources/capacity-pools/index).

Lane advice sets temporary running or unsettled caps. Advice can provide a lifetime in seconds or an explicit expiry; a lifetime defaults to 300 seconds. A running cap of `0` pauses admission until the advice expires or is deleted.

Advice with the exact partition tag applies to one lane. Advice submitted without tags can apply across the destination's partitioned lanes.

## Cardinality

A destination can hold at most **50,000 lane identities**. Ingress rejects a new identity beyond that limit with `TAG_CARDINALITY_EXCEEDED`; existing identities continue to accept work.

Request identifiers, timestamps, and other unbounded values should stay outside lane identity. A partition tag can keep descriptive tags on the action while bounding lane identity to the selected values.
