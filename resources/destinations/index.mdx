A destination is where an action is handed off for processing. Link at least one destination to an action definition, then use a default or [rules](/resources/rules/index) to choose between several.

The settlement boundary depends on the destination. Webhooks and SDK listeners can acknowledge work after processing it. Pub/Sub settles when Google Cloud accepts the publication; a separate subscriber's completion is outside that handoff.

Every destination starts with its own [capacity pool](/resources/capacity-pools/index). Group destinations only when they draw from the same underlying worker fleet, deployment, quota, or other constrained resource.
